# macOS SSH セキュリティ強化ガイド

このディレクトリには、macOS の SSH サーバー機能を安全に運用するための設定テンプレートが含まれています。

## 含まれているファイル

- `custom-security.conf`: セキュリティを最大化した SSH 設定のテンプレート。

## 導入手順

### 1. 公開鍵認証の準備
iPhone (Termius) などで作成した公開鍵を、Mac の `~/.ssh/authorized_keys` に登録してください。
**※この設定を適用すると、パスワードでのログインができなくなります。**

### 2. テンプレートの編集
`custom-security.conf` を開き、以下のプレースホルダーをご自身の環境に合わせて書き換えてください。

- `SSH_PORT`: 1024〜65535 の間の任意の数字（例: 49407）
- `SSH_ALLOW_USERNAME`: あなたの Mac のユーザー名（`whoami` コマンドで確認可能）

### 3. 設定ファイルの配置
編集したファイルを macOS の SSH 設定ディレクトリにコピーします。

```bash
sudo cp docs/custom-security.conf /etc/ssh/sshd_config.d/
```

### 4. リモートログインの有効化
システムのリモートログインを ON にします。

```bash
sudo systemsetup -setremotelogin on
```

### 5. 設定のテストと反映
設定ファイルの構文にエラーがないか確認し、SSH サービスを再起動して反映させます。

```bash
# 構文チェック
sudo sshd -t

# 再起動
sudo launchctl unload /System/Library/LaunchDaemons/ssh.plist
sudo launchctl load -w /System/Library/LaunchDaemons/ssh.plist
```

## セキュリティ効果

この設定を適用することで、以下の攻撃から Mac を保護します。

1.  **ポートスキャンの回避**: デフォルトの 22 番ポートを使用しないため、自動攻撃の対象外になります。
2.  **総当たり攻撃（ブルートフォース）の完全遮断**: パスワード認証を禁止し、鍵認証のみを許可します。
3.  **不正な対話形式ログインの防止**: チャレンジ・レスポンス認証およびキーボード対話認証を無効化します。
4.  **放置セッションの保護**: 5分間の無通信で自動切断します。
5.  **最新の暗号方式への限定**: 脆弱性が懸念される古い暗号方式（AES-CBCやSHA1等）を排除し、Curve25519 や Chacha20 等の現代的で強力なアルゴリズムのみを許可します。
6.  **踏み台化の防止**: Agent Forwarding や TCP Forwarding を禁止し、万が一侵入された際の被害拡大（他のサーバーへの横展開）を防ぎます。
7.  **詳細なログ記録**: ログレベルを VERBOSE に設定し、不正アクセスの試行を詳細に記録します。
8.  **厳格な権限チェック**: StrictModes により、設定不備（鍵ファイルの権限不足など）がある場合に接続を拒否し、セキュリティホールを未然に防ぎます。
9.  **IPv4 通信への限定**: AddressFamily inet により、未対策になりがちな IPv6 経路を完全に封鎖し、不要な IPv6 関連のシステムエラーも抑制します。

## 注意事項
- ポート番号を変更した場合、クライアント側（Termius等）のポート設定も必ず変更してください。
- ファイルを編集・移動する際は、常に `sudo` 権限が必要です。
- 古い SSH クライアントを使用している場合、最新の暗号方式のみを許可しているため接続できない可能性があります。その場合は最新のクライアントを使用してください。
